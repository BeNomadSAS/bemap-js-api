Notes

What this exercises

  • One 5000-point route (synthetic sine wave through France).
  • Coloured in 10 segments — traffic-style (green/yellow/red).
  • Slider moves a marker along it + highlights the traversed portion.
  • FPS is computed over the last 30 frames during scrub.

NAIVE mode

On every slider tick: removeMarker → new bemap.Icon → new bemap.Marker → addMarker. In MapLibre each add/remove is a DOM mutation + paint. This is what evmove5's chart-hover scrubber does today.

FAST mode

Marker built once. Each tick = marker.setCoordinate(c) (one setLngLat) + setPaintProperty('line-gradient', …). No DOM churn, no source mutation. This is the bfleet equivalent of OL's cheap vector-feature add/remove.

API used

marker.setCoordinate setLngLat
line-gradient paint property
lineMetrics true
